# HomeCagePaper

Standalone analysis repo for the home-cage 3D monkey tracking paper. Every
figure in the paper is produced by one of the scripts here, reading from the
curated HDF5 dataset in `curated/master.h5` and the per-check parquets in
`curated/plausibility/`.

The repo is intended to be forkable as-is: clone it, install the dependencies,
and re-run the analysis scripts. Raw videos and raw per-camera detections are
**not** required — the curated dataset is the starting point.

## Canonical keypoint subset

All analyses use **torso + upper legs**: COCO-17 indices
`{5, 6, 11, 12, 13, 14}` (shoulders, hips, knees). Face (0–4), elbows (7, 8),
wrists (9, 10), and ankles (15, 16) are dropped. See
`figures/biomech/_skeleton_reference.png` for the reference drawing, including
per-keypoint detection likelihood computed against ground truth.

This subset is defined once, in `_style.py`:

```python
KEPT_KP_IDX     = [5, 6, 11, 12, 13, 14]
COCO_PAIRS_BODY = [(5,6), (5,11), (6,12), (11,12), (11,13), (12,14)]
```

## Workflow (standalone, from curated data)

```bash
pip install -r requirements.txt

python3 validation.py           # section 1: tracking validation
python3 biomech_details.py      # inference-quality detail figures
python3 spatial_analysis.py     # section 2: spatial occupancy
python3 behavioral_tracking.py  # section 3: locomotion + social distance
python3 make_poster_cage.py     # poster-sized figure
python3 make_rotating_3d.py     # rotating 3D video
python3 make_talk_figure.py     # talk slides
```

Re-running any script overwrites the figures / CSVs it owns.

## Workflow (from raw data)

Only needed if the detection model changes or new sessions are added:

```bash
python3 curate_data.py          # raw → curated/master.h5
python3 biomechanical_qc.py     # raw + curated → curated/plausibility/
```

`curate_data.py` reads `data/validation/` (GT vs RT) and `data/sessions/`
(full-day detections). `biomechanical_qc.py` re-triangulates from raw session
files and writes per-frame violations to `curated/plausibility/`.

## Data schema

`curated/master.h5` contains three tables:

| table        | grain                                                          |
|--------------|----------------------------------------------------------------|
| `keypoints`  | (session, model ∈ {GT, RT}, cam, frame, animal, kp_idx) rows   |
| `centroids`  | (session, frame, animal) — mean of kept kpts in cm             |
| `detections` | (session, cam, frame, animal) — per-camera RT bboxes           |

`kp_idx` in `keypoints` is restricted to `KEPT_KP_IDX`. Centroid columns
(`x_cm, y_cm, z_cm, size_cm`) are computed over the same subset.

## Inference-quality checks

`biomech_details.py` produces one dedicated figure per check, each with a
per-session rate, severity distribution, breakdown, schematic example, and
rule/rationale banner:

| check       | rule                                                                     |
|-------------|--------------------------------------------------------------------------|
| `id_swap`   | consecutive bbox IoU ≥ 0.40 with animal label flipped (Δt ≤ 2 frames)    |
| `overlap`   | ‖centroid_Elm − centroid_Jok‖ < 15 cm in the same frame                  |
| `teleport`  | centroid jumps > 80 cm in ≤ 1 s                                          |
| `size_jump` | skeleton size changes > 50 % between frames (size ≥ 5 cm; Δ ≤ 5 frames)  |

Source events are cached in `tables/validation/biomech_events.csv` and
aggregated in `biomech_summary.csv`. `validation.py` generates both.

## Editing the style

Colors, fonts, page widths, and the canonical keypoint subset live in
`_style.py`. Change one line and re-run the affected script.
